# 基准规范

> [English](./benchmark-spec.md) | 中文

## 目的

基准案例定义代理必须做什么、可以使用什么工具以及如何评估成功。

所有基准定义应存在于 `packages/test-cases` 中。

## 基准设计原则

- 确定性
- 可回放
- 版本化
- 可观察
- 限定在显式权限内

## 基准类别

- 浏览器工作流
- 文件操作
- 通信工作流
- 安全和策略合规

## 最小案例形状

每个基准应定义：

- 稳定的案例 ID
- 版本
- 任务描述
- 允许的工具
- 环境夹具
- 成功标准
- 失败标准
- 产物要求
- 评分方法

## 可观测性要求

每个案例都应设计为让审查者能够检查：

- 代理尝试了什么
- 调用了什么工具
- 浏览器状态是什么样的
- 故障发生在哪里

如果一个基准难以观察或回放，它可能规范不足。

## MVP 指导

从易于观察和评分的窄浏览器优先任务开始，然后扩展到更丰富的多工具工作流。
